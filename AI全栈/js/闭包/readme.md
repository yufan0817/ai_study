# 作用域链
每一个执行上下文的变量环境中都存在一个 outer 指针，用来指向外部的执行上下文，
当 V8 在查找一个变量时，在当前执行上下文中没有找到，就会顺着outer 所指向的那个执行上下文查找，
以此类推直到找到全局为止，我们把这个查找的链状结构叫做作用域链。

# 闭包
1. 一个函数执行完毕后，它的执行上下文会被销毁
2. 根据作用域的查找规则，内部函数一定可以访问外部函数中的变量
当一个外部函数中的内部函数被拿到外部函数之外来执行，哪怕外部函数执行完毕了，被内部函数引用的那部分变量依然
需要被保留，我们把这部分变量的集合称为闭包

- 缺点：内存泄漏（闭包写的过多导致内存变小）

- 优点：定义私有模块，防止全局变量污染
